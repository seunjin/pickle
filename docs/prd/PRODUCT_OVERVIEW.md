# 🥒 Pickle 제품 개요 (Product Overview)

> **웹의 모든 영감을 한곳에, 피클.**

Pickle은 웹 서핑 중 발견한 텍스트, 이미지, 캡처, 북마크를 가장 빠르고 안전하게 보관할 수 있는 **개인 라이브러리 서비스**입니다.

---

## 1. 제품 비전 (Vision)

웹에서 발견하는 순간을 놓치지 않고 저장하여, 나만의 지식 저장소를 구축할 수 있도록 돕는 것이 Pickle의 핵심 목표입니다.

### 핵심 가치 (Core Values)
- **즉시성**: 단축키 한 번으로 현재 보고 있는 콘텐츠를 즉시 저장
- **통합성**: 텍스트, 이미지, 화면 캡처, 북마크를 하나의 라이브러리에서 관리
- **접근성**: 브라우저 익스텐션과 웹 대시보드 간 실시간 동기화
- **보안성**: 개인 데이터는 본인만 접근 가능 (RLS 기반 접근 제어)

## 2. 타겟 사용자 (Target Users)

| 사용자 유형 | 설명 | 주요 사용 시나리오 |
|:---|:---|:---|
| **리서처** | 자료 수집이 많은 사용자 | 아티클 북마크, 웹 캡처 저장 |
| **디자이너** | 영감을 모으는 크리에이터 | UI 스크린샷, 레퍼런스 이미지 수집 |
| **학생/연구자** | 학습 자료를 정리하는 사용자 | 웹 자료 스크랩, 인용문 저장 |
| **일반 사용자** | 웹 브라우징 중 북마크 대체 필요 | 페이지 북마크, 관심 콘텐츠 저장 |

## 3. 플랫폼 구성 (Platform Structure)

```
┌─────────────────────────────────────────────────────────────┐
│                     Pickle 서비스                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐          ┌─────────────────────────┐  │
│  │  Chrome/Whale   │◀────────▶│     Web Dashboard       │  │
│  │   Extension     │  Sync    │    (picklenote.io)      │  │
│  └─────────────────┘          └─────────────────────────┘  │
│           │                              │                  │
│           └──────────┬───────────────────┘                  │
│                      ▼                                      │
│            ┌─────────────────────┐                          │
│            │   Supabase Backend  │                          │
│            │  (Auth, DB, Storage)│                          │
│            └─────────────────────┘                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.1 브라우저 익스텐션
- **지원 브라우저**: Chrome, Whale (추후 Firefox, Edge 확장 예정)
- **핵심 역할**: 웹 페이지에서 콘텐츠를 빠르게 캡처/저장
- **동작 방식**: 단축키 기반 즉시 저장, 백그라운드 DB 직접 통신

### 3.2 웹 대시보드
- **URL**: picklenote.io
- **핵심 역할**: 저장된 노트의 조회, 검색, 정리, 관리
- **기술 스택**: Next.js (App Router), React 19, Supabase

## 4. 핵심 기능 (Key Features)

### 4.1 콘텐츠 저장 (Content Capture)

| 기능 | 설명 | 트리거 |
|:---|:---|:---|
| **Quick Capture** | 화면의 특정 영역을 드래그하여 이미지로 저장 | 단축키 |
| **이미지 저장** | 웹 페이지 내 이미지를 직접 저장 | 우클릭 메뉴 |
| **텍스트 저장** | 선택한 텍스트를 메모와 함께 저장 | 단축키 (텍스트 선택 상태) |
| **북마크** | 페이지 메타데이터(OG Tag, 파비콘)를 자동 추출하여 저장 | 단축키 |

### 4.2 콘텐츠 관리 (Content Management)

| 기능 | 설명 |
|:---|:---|
| **대시보드** | 저장된 모든 노트를 한눈에 확인 |
| **검색** | 제목, 메모, URL, 태그 기반 전문 검색 |
| **태그** | 노트에 태그를 부여하여 분류 |
| **폴더** | 노트를 폴더별로 정리 |
| **북마크** | 중요 노트를 별도로 관리 |
| **휴지통** | 삭제된 노트를 임시 보관 및 복구 |

### 4.3 사용자 계정 (User Account)

| 기능 | 설명 |
|:---|:---|
| **소셜 로그인** | Google 계정으로 간편 가입/로그인 |
| **약관 동의** | 서비스 이용약관, 개인정보 처리방침, 마케팅 동의 |
| **프로필 관리** | 이름, 프로필 사진 변경 |
| **회원 탈퇴** | 즉시 계정 및 데이터 삭제 |

## 5. 데이터 모델 개요 (Data Model)

### 5.1 노트 유형 (Note Types)

| Type | 설명 | 저장 데이터 |
|:---|:---|:---|
| `text` | 텍스트 스크랩 | 선택 텍스트, 원본 URL, 메타데이터 |
| `image` | 이미지 저장 | 이미지 파일(Asset), 원본 URL, 메타데이터 |
| `capture` | 화면 캡처 | 캡처 이미지(Asset), 캡처 크기, 원본 URL |
| `bookmark` | 페이지 북마크 | URL, OG 메타데이터(제목, 설명, 이미지) |

### 5.2 워크스페이스 (Workspace)
- 데이터가 저장되는 최상위 컨테이너
- 가입 시 기본 워크스페이스("내 워크스페이스") 자동 생성
- 향후 팀 협업을 위한 멀티 워크스페이스 지원 예정

## 6. 기술적 특징 (Technical Highlights)

| 영역 | 기술 |
|:---|:---|
| **인증** | Supabase Auth + Google OAuth (PKCE) |
| **데이터베이스** | PostgreSQL (Supabase) + RLS |
| **파일 스토리지** | Supabase Storage (이미지 최적화 지원) |
| **프론트엔드** | React 19, Next.js App Router, TanStack Query |
| **익스텐션** | Chrome MV3, Vite, React |
| **모노레포** | pnpm workspaces + Turborepo |
