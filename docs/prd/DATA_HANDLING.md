# 📊 Pickle 데이터 취급 명세

이 문서는 Pickle 서비스의 데이터 취급 정책을 명세합니다. **이용약관** 및 **개인정보 처리방침** 작성 시 참고.

---

## 1. 수집 데이터

### 1.1 계정 정보

| 항목 | 출처 | 필수 | 보관 |
|:---|:---|:---|:---|
| 이메일 | Google OAuth | ✅ | 탈퇴 시 |
| 이름 | Google OAuth | ✅ | 탈퇴 시 |
| 프로필 사진 | Google OAuth | ❌ | 탈퇴 시 |

### 1.2 동의 정보

| 항목 | 필수 |
|:---|:---|
| 서비스 이용약관 | ✅ |
| 개인정보 처리방침 | ✅ |
| 마케팅 수신 | ❌ |

### 1.3 사용자 콘텐츠

| 항목 | 설명 |
|:---|:---|
| 텍스트 노트 | 저장한 텍스트 |
| 이미지/캡처 | 저장한 이미지 파일 |
| 북마크 | 페이지 URL/메타데이터 |
| 태그/폴더 | 분류 정보 |
| 메모 | 사용자 메모 |

### 1.4 자동 수집 메타데이터

- 원본 페이지 URL
- 페이지 제목, 설명
- 파비콘, OG 이미지
- 생성/수정 일시

---

## 2. 수집 방식

| 경로 | 데이터 | 동의 |
|:---|:---|:---|
| Google OAuth | 이메일, 이름 | OAuth 동의 |
| 익스텐션 단축키 | 선택 콘텐츠, 메타데이터 | 명시적 트리거 |

> **중요**: 모든 콘텐츠 저장은 사용자의 명시적 단축키 입력으로만 발생합니다.

---

## 3. 저장 및 보안

| 데이터 | 저장 위치 | 암호화 |
|:---|:---|:---|
| 계정 정보 | Supabase Auth | SSL/TLS |
| 노트 데이터 | Supabase DB | SSL/TLS |
| 파일 | Supabase Storage | SSL/TLS |

- **RLS**: 본인 데이터만 접근 가능
- **용량 제한**: 워크스페이스당 50MB

---

## 4. 삭제 정책

### 노트 삭제
1. 휴지통 이동 (Soft Delete) → 복구 가능
2. 영구 삭제 → 복구 불가

### 회원 탈퇴
- **즉시 삭제**: 계정, 프로필, 노트, 태그, 폴더, 파일 전체
- **Cascade Delete**: 연관 데이터 자동 삭제
- **복구 불가**

## 5. 제3자 공유

| 서비스 | 목적 |
|:---|:---|
| Google | OAuth 인증 |
| Supabase | 인프라 제공 |

> 사용자 콘텐츠는 제3자와 공유하지 않습니다.
