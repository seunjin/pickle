# 📌 Executive Summary

## 한 줄 요약
> **Pickle**은 웹 서핑 중 발견한 콘텐츠를 단축키 한 번으로 저장하고, 언제든 다시 찾아볼 수 있는 개인 라이브러리 서비스입니다.

## 핵심 가치 제안

| 💡 Pain Point | ✨ Pickle의 해결책 |
|:---|:---|
| "이 글 나중에 읽어야지..." → 잊어버림 | 텍스트 드래그 + 단축키로 하이라이트 저장 |
| "북마크가 너무 많아 못 찾겠어" | 검색, 태그, 폴더로 체계적 관리 |
| "스크린샷 찍고 어디 저장했지?" | 화면 캡처 + 자동 클라우드 동기화 |

# 🎯 1. 제품 비전 및 목표

## 1.1 제품 비전

```
"웹의 모든 영감을 한곳에, 피클."
```

우리는 웹에서 발견하는 모든 순간을 놓치지 않고 저장하여, 사용자만의 **디지털 영감 라이브러리**를 구축할 수 있도록 돕습니다.

## 1.2 제품 목표

### 🚀 핵심 목표

| 목표 | 설명 | 측정 지표 |
|:---|:---|:---|
| **즉시성** | 단축키 한 번으로 즉시 저장 | 저장 완료까지 3초 이내 |
| **통합성** | 모든 유형의 콘텐츠를 하나로 | 3가지 저장 유형 (텍스트/캡처/북마크) |
| **접근성** | 어디서든 내 콘텐츠에 접근 | 익스텐션 ↔ 웹 실시간 동기화 |
| **보안성** | 내 데이터는 나만 볼 수 있음 | RLS 기반 100% 데이터 격리 |

### 📊 비즈니스 목표

| 지표 | 3개월 목표 | 6개월 목표 |
|:---|:---|:---|
| MAU (월간 활성 사용자) | 3,000명 | 10,000명 |
| 익스텐션 설치 수 | 5,000건 | 20,000건 |
| 일일 저장 노트 수 | 평균 5개/사용자 | 평균 10개/사용자 |
| 30일 리텐션 | 40% | 60% |

**우선순위**: P0 (필수)  
**담당**: Extension

| ID | 요구사항 | 수락 기준 |

| SAVE-001-3 | 단축키 트리거 | 기본: Cmd + N |
| SAVE-001-4 | 저장 피드백 | 토스트로 성공/실패 알림 |

#### 🔄 사용자 플로우

```
1. 사용자: 웹페이지에서 텍스트 드래그 선택
2. 사용자: Cmd + N 입력
3. 익스텐션: 선택 텍스트 + 메타데이터 추출
4. 익스텐션: 저장 요청 → 백그라운드
5. 백그라운드: DB에 노트 저장
6. 사용자: "저장되었습니다" 토스트 확인
```

| 필드 | 설명 | 예시 |
|:---|:---|:---|
| `type` | 노트 유형 | `text` |
| `data.text` | 선택한 텍스트 | "이것은 선택한 텍스트입니다" |
| `url` | 원본 페이지 URL | https://example.com/article |
| `meta.title` | 페이지 제목 | "예제 아티클" |
| `meta.favicon` | 파비콘 URL | https://example.com/favicon.ico |

# ⚙️ 5. 비기능 요구사항 (Non-Functional Requirements)

## 5.1 성능 (Performance)

| 항목 | 요구사항 | 측정 방법 |
|:---|:---|:---|
| 저장 응답 시간 | 3초 이내 | 단축키 입력 → 성공 토스트 |
| 대시보드 초기 로딩 | 2초 이내 (LCP) | Lighthouse |
| 검색 응답 시간 | 1초 이내 | 검색어 입력 → 결과 표시 |
| 무한 스크롤 로딩 | 500ms 이내 | 스크롤 트리거 → 추가 로딩 |

## 5.2 보안 (Security)

| 항목 | 요구사항 |
|:---|:---|
| 전송 암호화 | 모든 통신 HTTPS (TLS 1.2+) |
| 인증 | OAuth 2.0 + PKCE |
| 데이터 격리 | RLS 기반 사용자별 데이터 접근 제한 |
| 세션 관리 | Access Token 1시간 만료, Silent Refresh |

## 5.3 호환성 (Compatibility)

| 플랫폼 | 지원 범위 |
|:---|:---|
| 익스텐션 브라우저 | Chrome 88+, Whale 최신 |
| 웹 대시보드 브라우저 | Chrome, Safari, Firefox, Edge 최신 |
| 웹 대시보드 해상도 | 1024px 이상 (반응형) |

## 5.4 가용성 (Availability)

| 항목 | 목표 |
|:---|:---|
| 서비스 가용성 | 99.5% 이상 |
| 데이터 내구성 | 99.99% (Supabase 보장) |
